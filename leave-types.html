Leave Types

The Leave Types page allows HR administrators to manage different categories of leave available in the system. This includes configuring default allocations, approval requirements, and other parameters for each leave type.

Accessing Leave Types

  1. Log in to the system with HR administrator credentials
  2. Navigate to the Branch Dashboard
  3. Click on "Leave" in the sidebar menu under the Organization section
  4. Click on "Types" from the submenu
  5. The Leave Types page will display all configured leave types

Leave Types List

The page displays a table with the following information for each leave type:

Table Columns:

  • Leave Type - Name of the leave category (e.g., Annual Leave, Sick Leave)
  • Code - Short identifier for the leave type (e.g., ANNUAL, SICK)
  • Default Days/Year - Number of days allocated by default to employees
  • Paid - Whether the leave is paid or unpaid
  • Requires Approval - Whether manager approval is required
  • Status - Whether the leave type is active or inactive
  • Actions - Available actions for each leave type

Filtering and Searching

Use the filtering options to find specific leave types:

Search Options:

  • Search Box - (text input) Enter text to search across leave type names, codes, and descriptions

Creating a New Leave Type

Add a new category of leave to the system:

Step-by-Step Creation Process:

  1. Click the "Create Leave Type" button (the button may appear as text, a plus icon, or similar depending on your interface) on the page
  2. A modal will appear with the leave type creation form
  3. Fill in the required fields (see below for details)
  4. Click the "Save" button (the button may appear as text, an icon, or similar depending on your interface) to create the new leave type
  5. The new leave type will appear in the list

Required Fields:

  • Name - (text input) Enter a descriptive name for the leave type (e.g., "Annual Leave", "Sick Leave")
  • Code - (text input) Enter a unique code for the leave type (e.g., "ANNUAL", "SICK") - will be converted to uppercase
  • Default Days/Year - (number input) Enter the default number of days allocated per year (must be 0 or greater)

Optional Fields:

  • Description - (textarea) Provide a detailed description of the leave type and its usage
  • Requires Approval - (checkbox) Check if manager approval is required for this leave type (default: checked)
  • Requires Document - (checkbox) Check if supporting documents are required (e.g., medical certificate for sick leave)
  • Max Consecutive Days - (number input) Maximum number of consecutive days that can be taken (optional)
  • Min Notice Days - (number input) Minimum number of days advance notice required (default: 0)
  • Is Paid - (checkbox) Check if this is paid leave (default: checked)
  • Is Active - (checkbox) Check if this leave type is currently available for use (default: checked)
  • Color - (color picker) Select a color to represent this leave type in calendars and reports

Editing a Leave Type

Modify an existing leave type configuration:

Step-by-Step Editing Process:

  1. Find the leave type you want to edit in the list
  2. Click the "Edit" button (the button may appear as text, a pencil icon, or similar depending on your interface) in the Actions column
  3. A modal will appear with the current values pre-filled
  4. Modify the fields as needed
  5. Click the "Save" button (the button may appear as text, an icon, or similar depending on your interface) to update the leave type
  6. The changes will be reflected in the list

Important Considerations:

  • Changes to default days will only affect new allocations, not existing ones
  • Disabling a leave type will prevent new applications but won't affect existing ones
  • Be cautious when changing codes as they may be referenced in reports

Toggling Leave Type Status

Activate or deactivate a leave type without deleting it:

Activation Process:

  1. Find the leave type in the list
  2. Click the "Toggle Status" button (the button may appear as text, a toggle switch, or similar depending on your interface) in the Actions column
  3. If the leave type was active, it will be deactivated; if inactive, it will be activated
  4. A confirmation message will appear indicating the new status

Status Impact:

  • Active leave types are available for employees to apply for
  • Inactive leave types are hidden from the application form
  • Existing applications for inactive types remain unaffected
  • Previously allocated days for inactive types remain available

Deleting a Leave Type

Remove a leave type from the system:

Deletion Process:

  1. Find the leave type you want to delete in the list
  2. Click the "Delete" button (the button may appear as text, a trash icon, or similar depending on your interface) in the Actions column
  3. A confirmation dialog will appear asking if you're sure you want to delete the leave type
  4. Click "Confirm" to delete the leave type or "Cancel" to keep it

Important Restrictions:

  • You cannot delete a leave type that has existing leave applications
  • Instead of deleting, consider deactivating the leave type
  • Deletion is permanent and cannot be undone
  • All associated data will be removed from the system

Leave Type Configuration Details

Default Days Per Year:

  • Defines the standard allocation for new employees
  • HR administrators can override this for individual employees
  • Allocations are typically reset annually

Approval Requirements:

  • When enabled, all applications for this leave type require manager approval
  • When disabled, applications may be auto-approved or not require approval
  • Provides control over specific types of leave

Document Requirements:

  • When enabled, applicants must upload supporting documents
  • Useful for sick leave, compassionate leave, or other verified absences
  • Documents are stored securely and accessible to authorized personnel

Consecutive Day Limits:

  • Prevents employees from taking extended periods off in a single request
  • Helps with operational planning and coverage
  • Does not prevent multiple shorter requests

Notice Requirements:

  • Requires employees to submit requests in advance
  • Helps with planning and coverage arrangements
  • Prevents last-minute scheduling disruptions

Pagination

The leave types list is divided into pages to improve performance. Use the pagination controls at the bottom of the table to navigate between pages. You can also adjust the number of leave types displayed per page using the quantity selector.

Tips and Best Practices

  • Define clear categories that align with company policies
  • Use descriptive names and codes that are easily understood
  • Set appropriate default allocations based on employment contracts
  • Configure approval requirements based on leave type sensitivity
  • Establish document requirements for verification purposes
  • Set reasonable consecutive day limits to balance flexibility with planning
  • Implement appropriate notice periods for different leave types
  • Use colors consistently across different leave types
  • Regularly review and update leave types to reflect policy changes
  • Ensure all stakeholders understand the different leave types and their requirements
