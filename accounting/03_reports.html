Standard accounting reports generated from GL entries

Overview

SweetTooth provides five standard financial reports generated from GL entries. Reports are defined in app/Services/Reports/Definitions/ and accessed through Livewire components.

Report Engine: Reports use TrialBalanceService for data generation
Location: app/Services/Reports/Definitions/Accounting*.php
Routes: All under /accounting/reports/ prefix

Available Reports

BALANCE

Balance Sheet

Snapshot of assets, liabilities, and equity at a specific date.

Route: /accounting/reports/balance-sheet

INCOME

Income Statement

Revenue, expenses, and net profit over a period.

Route: /accounting/reports/income-statement

TRIAL

Trial Balance

All accounts with debit/credit balances. Validates posting accuracy.

Route: /accounting/reports/trial-balance

LEDGER

General Ledger

Detailed transaction listing for all accounts or specific account.

Route: /accounting/reports/general-ledger

CASH

Cash Flow Statement

Cash movements categorized by operating, investing, financing.

Route: /accounting/reports/cash-flow

Report Definitions

AccountingTrialBalanceDefinition

File: app/Services/Reports/Definitions/AccountingTrialBalanceDefinition.php

Purpose: Generate trial balance showing all accounts with debit/credit balances.

Query Parameters

  • branch_id - Filter by branch (optional)
  • period_from - Start date for reporting period
  • period_to - End date for reporting period

Output Structure

  • summary: total_debits, total_credits, is_balanced, difference
  • accounts: Array of accounts with account_number, account_name, type, debit, credit
  • period_info: from, to dates

Validation

  • Trial balance is balanced when total_debits = total_credits
  • Difference field shows imbalance amount if not balanced

AccountingBalanceSheetDefinition

File: app/Services/Reports/Definitions/AccountingBalanceSheetDefinition.php

Purpose: Report assets, liabilities, and equity at period end.

Sections

  • Assets: Current Assets, Fixed Assets, Other Assets
  • Liabilities: Current Liabilities, Long-term Liabilities
  • Equity: Owner's Equity, Retained Earnings, Current Year Earnings

Accounting Equation

Assets = Liabilities + Equity

AccountingIncomeStatementDefinition

File: app/Services/Reports/Definitions/AccountingIncomeStatementDefinition.php

Purpose: Report revenue, expenses, and net income for a period.

Sections

  • Revenue: Sales Revenue, Other Income
  • COGS: Cost of Goods Sold
  • Gross Profit: Revenue - COGS
  • Operating Expenses: Salaries, Rent, Utilities, etc.
  • Net Income: Gross Profit - Operating Expenses

Formula

Net Income = Revenue - COGS - Operating Expenses

AccountingGeneralLedgerDefinition

File: app/Services/Reports/Definitions/AccountingGeneralLedgerDefinition.php

Purpose: Detailed listing of all transactions for accounts.

Query Parameters

  • account_id - Specific account (optional, all if not specified)
  • branch_id - Filter by branch
  • date_from - Start date
  • date_to - End date

Output Columns

  • Date, Reference Number, Description, Entry Type
  • Debit, Credit, Running Balance

AccountingCashFlowStatementDefinition

File: app/Services/Reports/Definitions/AccountingCashFlowStatementDefinition.php

Purpose: Track cash movements by activity type.

Sections

  • Operating Activities: Cash from sales, payments to suppliers, operating expenses
  • Investing Activities: Asset purchases/sales, investments
  • Financing Activities: Loans, owner contributions, distributions

Reconciliation

  • Beginning Cash Balance
  • + Net Cash from Operating
  • + Net Cash from Investing
  • + Net Cash from Financing
  • = Ending Cash Balance

Report Generation Flow

REPORT GENERATION FLOW: 1. USER REQUESTS REPORT ‚Üì 2. LIVEWIRE COMPONENT RECEIVES PARAMETERS - branch_id, period_from, period_to, account filters ‚Üì 3. REPORT DEFINITION CALLED ‚Üì 4. TRIALBALANCESERVICE::GENERATE() - Queries GlEntry records - Groups by gl_account_id - Calculates debit/credit totals - Joins with GlAccount for names ‚Üì 5. DEFINITION::SUMMARY() CALLED - Computes totals, ratios, checks ‚Üì 6. DEFINITION::TABLES() CALLED - Formats data for display ‚Üì 7. DEFINITION::CHARTS() CALLED (optional) - Prepares visualization data ‚Üì 8. BLADE VIEW RENDERS - Displays tables, charts, narrative

TrialBalanceService

The TrialBalanceService is the core data engine for all financial reports.

Key Method

public function generate( ?int $departmentId = null, ?Carbon $from = null, ?Carbon $to = null, ?int $branchId = null ): array { // 1. Get all GL accounts $accounts = GlAccount::with('glEntries')->get(); // 2. Filter entries by date range if ($from) { $query->where('entry_date', '>=', $from); } if ($to) { $query->where('entry_date', '<=', $to); } // 3. Calculate balances per account foreach ($accounts as $account) { $debit = $account->glEntries->sum('debit'); $credit = $account->glEntries->sum('credit'); $result[] = [ 'account_number' => $account->account_number, 'account_name' => $account->name, 'account_type' => $account->type, 'debit' => $debit, 'credit' => $credit, 'balance' => $debit - $credit, ]; } // 4. Compute summary $summary = [ 'total_debits' => array_sum(array_column($result, 'debit')), 'total_credits' => array_sum(array_column($result, 'credit')), 'is_balanced' => $total_debits === $total_credits, 'difference' => abs($total_debits - $total_credits), ]; return ['accounts' => $result, 'summary' => $summary]; }

Report Validation

Trial Balance Checks

  • Balanced: Total Debits = Total Credits
  • Difference: Should be 0.00
  • All Accounts: Every GL account should appear

Balance Sheet Checks

  • Equation: Assets = Liabilities + Equity
  • Retained Earnings: Should link to prior period net income
  • Current Year Earnings: Should match Income Statement net income

Income Statement Checks

  • Gross Profit: Revenue - COGS
  • Net Income: Gross Profit - Operating Expenses
  • Period Match: Net income should match equity change
Reconciliation Tip: Run Trial Balance first to verify data integrity. If trial balance doesn't balance, investigate GL entries before running other reports.

Using Reports

Step-by-Step

  1. Navigate to Accounting ‚Üí Reports
  2. Select desired report type
  3. Set date range or accounting period
  4. Optionally filter by branch or department
  5. Click Generate/Run
  6. Review results, export if needed

Export Options

  • PDF - For printing and formal distribution
  • Excel - For further analysis and manipulation
  • CSV - For importing to other systems
