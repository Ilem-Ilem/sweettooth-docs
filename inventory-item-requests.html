Creating Item Requests

Process Overview

Creating item requests involves specifying the department, items needed, and quantities required.

Required Information:

  • Department: Select the requesting department
  • Request Date: Date when items are needed (defaults to tomorrow)
  • Items: List of items to be requested with quantities
  • Notes: Additional information about the request

Steps to Create an Item Request:

  1. Navigate to the Item Requests section
  2. Click the "Add New Request" button
  3. Select the requesting department from the dropdown
  4. Set the required date for the items
  5. Add items to the request using the "Add Item" button
  6. Select the item from the inventory list
  7. Specify the quantity needed
  8. Repeat for all required items
  9. Add any relevant notes about the request
  10. Click "Save" to submit the request

Item Selection:

  • Select items from the active inventory items list
  • Quantities must be positive numbers
  • The system validates stock availability before approval
  • Only items with sufficient stock can be requested

Tip: The system automatically generates a unique request number based on your branch code and department. This ensures proper tracking of all requests.

Important: The system validates that requested quantities do not exceed available stock. If stock is insufficient, the request will not be created and an error will be displayed.

Request Process

Process Overview

The item request process follows a structured workflow from creation to fulfillment.

Request Lifecycle:

  1. Creation: Department creates item request with required items and quantities
  2. Submission: Request is submitted for approval
  3. Review: Inventory team reviews the request and available stock
  4. Approval: Request is approved and items are reserved
  5. Fulfillment: Items are dispatched to the requesting department

Status Transitions:

  • Draft: Request is being prepared (if applicable)
  • Pending: Request submitted and awaiting approval
  • Approved: Request approved and items reserved
  • Partially Dispatched: Some items have been dispatched
  • Completed: All items have been dispatched

Request Validation:

  • System checks for sufficient stock before approval
  • Validates that requested quantities are positive
  • Ensures department has permission to request items
  • Verifies item status (active/inactive)

Best Practice: Plan requests in advance to ensure sufficient lead time for approval and fulfillment.

Managing Item Requests

Process Overview

Once requests are created, you can view details, track status, and manage related activities.

Viewing Request Details:

  • Click on a request in the list to view detailed information
  • See all items included in the request
  • View department information and requester details
  • Check the status of the request
  • Review quantities requested, approved, and dispatched

Request Management Activities:

  • Status Tracking: Monitor the progress of requests
  • Quantity Monitoring: Track approved vs. requested quantities
  • Dispatch Coordination: Coordinate with dispatch team
  • Communication: Notify departments of status changes

Actions Available:

  • View Details: See complete request information
  • Approve: Approve the request and reserve items (in Dispatch module)
  • Dispatch: Release items to requesting department (in Dispatch module)
  • Export: Download request data in Excel or CSV

Important: Requests must be approved before items can be dispatched. Approval happens in the Dispatch module.

Request Approvals

Process Overview

Request approvals ensure proper authorization for inventory allocation and prevent unauthorized usage.

Approval Process:

  1. Department creates item request and submits it
  2. Request appears in the pending queue in the Dispatch module
  3. Inventory team reviews the request and available stock
  4. Approve the request to reserve items from stock
  5. Approved items are reserved and ready for dispatch

Approval Requirements:

  • All requests must be approved before dispatch
  • Approvals are handled in the Dispatch module
  • Approving reserves items from available stock
  • Approvals are tracked in the audit log

What Happens After Approval:

  • Request status changes to "approved"
  • Requested items are reserved from available stock
  • Reserved quantities are deducted from available stock
  • Items become unavailable for other requests
  • Ready for dispatch to the requesting department

Tip: Review requests carefully to ensure they align with operational needs and available inventory.

Stock Reservation

Process Overview

Stock reservation occurs when requests are approved, ensuring items are set aside for the requesting department.

Reservation Process:

  1. Department requests specific items and quantities
  2. Request is submitted and awaits approval
  3. Upon approval, requested quantities are reserved
  4. Reserved quantities are deducted from available stock
  5. Reserved items cannot be used for other requests
  6. Items remain reserved until dispatched

Reservation Impact:

  • Reduces available stock for other departments
  • Prevents double-allocation of items
  • Ensures requested items will be available
  • Updates stock levels in real-time
  • Creates audit trail of reservations

Reservation Management:

  • Reservations are automatically created upon approval
  • Reservations are released when items are dispatched
  • Cancelled requests release reserved stock
  • Monitor reservation duration to prevent stock lockup

Best Practice: Monitor reservation duration to ensure timely dispatch and prevent unnecessary stock lockup.

Filters and Search

Overview

The Item Requests interface includes filtering and search capabilities to help you find specific requests quickly.

Available Filters:

  • Search: Search by request number, department name, or requester name
  • Department Filter: Filter by requesting department
  • Status Filter: Filter by request status (pending, approved, completed, etc.)

Using Filters:

  1. Use the search bar to find requests by number or department
  2. Select a department from the dropdown to filter by requesting department
  3. Select a status to filter by lifecycle stage
  4. Combine multiple filters for more specific results

Additional Features:

  • Sort requests by clicking on column headers
  • Adjust the number of requests displayed per page
  • Export filtered results to Excel or CSV
  • View request summary statistics

Permissions and Access

Overview

Access to Item Requests features is controlled by user roles and permissions.

Role-Based Access:

  • Super Admin: Full access to create, approve, and manage requests
  • Department Head: Can create requests for their department
  • Inventory Manager: Can manage and approve requests
  • Store Keeper: Can view and process requests
  • Regular Employee: May have read-only access depending on permissions

Permission Considerations:

  • Users can typically only create requests for their own department
  • Approval permissions are usually restricted to inventory managers
  • Higher-level roles can approve requests from any department
  • Some departments may have restricted access to certain items

Tip: Contact your system administrator if you need elevated permissions for request management tasks.

Best Practices

Optimizing Request Management

Follow these best practices to maintain efficient and effective request management.

Planning Requests:

  • Plan requests in advance to ensure sufficient lead time
  • Coordinate with production schedules and project timelines
  • Consider batch ordering to reduce administrative overhead
  • Align requests with budget cycles and financial planning

Request Accuracy:

  • Verify item availability before creating requests
  • Specify exact quantities needed to avoid waste
  • Include detailed specifications when necessary
  • Provide clear justifications for requests

Communication:

  • Communicate expected fulfillment dates to requesting departments
  • Notify departments of any delays or changes
  • Provide status updates throughout the process
  • Coordinate with dispatch team for timely fulfillment

Documentation:

  • Keep detailed records of all requests for auditing
  • Document reasons for any special handling requirements
  • Maintain logs of approved and denied requests

Pro Tip: Regularly review request patterns to identify opportunities for process improvements and efficiency gains.
