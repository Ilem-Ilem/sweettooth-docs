Tips & Tricks

Helpful tips and tricks to make the most out of the SweetTooth system.

General Tips

  • Modal Navigation: Most forms and detailed views open in modals. To close any modal without saving, click the close button (typically an "X" icon) in the top-right corner.
  • Confirmation Dialogs: Important actions like deletions will show confirmation dialogs. Always read these carefully before confirming.
  • Bulk Actions: You can select multiple items using checkboxes and perform actions on all selected items at once.
  • Search and Filter: Use the search bar and filter options to quickly find specific items in large lists.
  • Pagination: Large lists are divided into pages. Use the pagination controls at the bottom to navigate between pages.

Roles & Permissions Tips

  • Permission Assignment: When creating or editing a role, you can assign multiple permissions at once by checking the boxes next to each permission.
  • Permission Guard: Make sure the guard for permissions matches the guard for the role (usually "web").
  • Viewing Permissions: Use the eye icon to quickly view all permissions assigned to a specific role without editing it.
  • Creating New Permissions: You can create new permissions directly from the role creation/editing modal.

Branch Management Tips

  • Unique Codes: Each branch must have a unique code. This helps identify branches in reports and other parts of the system.
  • Soft Delete: When you delete a branch, it moves to the "Deleted Branches" section where it can be restored if needed.
  • Branch Managers: Assign a user as the branch manager to give them specific responsibilities for that location.
  • Geographic Information: Accurate country, state, and city information helps with reporting and analytics.

Troubleshooting

  • Can't See Menu Options: If you don't see certain menu options, you may not have the required permissions. Contact your system administrator.
  • Modal Won't Close: If a modal becomes unresponsive, try clicking the close button (often an "X" icon) or refresh the page to return to the main screen.
  • Permission Issues: If you receive "Access Denied" errors, verify that your role has the necessary permissions for the action you're trying to perform.
  • Data Not Saving: If form data isn't saving, check that all required fields are filled correctly and that there are no validation errors.

Best Practices

  • Regular Reviews: Regularly review roles and permissions to ensure users have appropriate access levels.
  • Clear Naming: Use clear, descriptive names for roles and branches to make management easier.
  • Documentation: Keep internal documentation of role responsibilities and branch information for reference.
  • Backup: Before making major changes to roles or deleting branches, ensure you have proper backups.
